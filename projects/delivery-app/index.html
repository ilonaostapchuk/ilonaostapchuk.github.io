Project

Manufacturers and dealers want to be able to ship the windows and door places. In order to do that they have two options: Use their truck and trailers, or use common carrier.

Currently, in order to have access to the truck's route, the driver must be a user in the manufacturer's ERP system.

Our customer does not want to create a new user to a one-time driver and the one-time driver does not want to install an app on their phone for a one-time job.

Challenge

Create a prototype for a mobile web server that can be accessed from a URL or a QR code to allow user to visualize shipment information.

My role

UX designer

User story #1

As a driver, I need to get the route information for the truck of products I'm delivering. Scan a QR code, barcode to load the driver information.

Ask for driver information Name + Cell phone number / Form could be pre-seed. Once the form is completed the user can navigate to the route information.

User story #2

As a driver, I want to easily see my route and each stop along the way. Allow an overview of the route with the drop pin on the route. Show distance and time between drops. Drop Stops Number of items to drop Access the list of product per drops

User story #3

As a driver, I want to notify the next drop site of the estimated time of arrival (ETA). Get ETA + allow to change the ETA: Have a message box to ask 'Send ETA to next stop'. Show the exact time map software indicates with a + / - that allows them to change by 15min increments When ETA is confirm then show directions.

User story #4

As a driver, I need to know and check the element that needs to get out of the truck. I normally need an image and some information about the window and door to unload (image, width, height, brief description). Allow scanning the product barcode to validate if it should be unloaded. Once a barcode is scanned, the application should show the product with the following information:
Scan bar instructions to 'Scan barcode to unload'
Sales Order number and Line number should be at the top of the
Image of the product
Description
Size Buttons: Unload or skip

User story #5

As a driver, I need a way to skip (not unload) an element that needs to be shipped. There are a couple of reasons why this can happen, I need to be able to specify what is the reason for not unloading the unit. I would need a way to upload one or more images and add any other comments that might be relevant. Note one or more images could get uploaded. There are a couple of reasons why a unit does not get unloaded, and only one should apply to the current unit. Multiple units could have the same skipped reason and comment drop down list of reasons (damaged, wrong product). Comment Field Option to take a picture Submit button

User story #6

As a driver, I need the customer signature before leaving the location. The driver can create signature requests. The signature request contains: The count of the products unloaded (example 15), The list of the skipped products - reason + picture + comments, The signature request is reviewed by the driver and final customer. The final customer signs it (options), On the driver's device, On his device (from a link in a text message or email, or from a QR code display on the driver cell phone ). The final customer only has access to his drop information

User story #7

As a driver, I need a reminder if I need to go back at the manufacturer site to unload skipped units. Final stop is the Mfg address Scan instructions change to 'Scan returned items' Button to Complete and Finish Shipment.
